# EZ데이터허브 데이터 정합성 강화 작업 메모 (Data Change Memo)

본 문서는 데이터허브 대시보드 리팩토링 과정에서 결정된 데이터 처리 규칙과 로직 변경 사항을 기록합니다.

---

### 1. 데이터 정규화 및 조인 규칙 (Normalization & Join)
*   **UserNo 표준화**: 모든 데이터의 `UserNo`는 다음 규칙을 거쳐 3자리 문자열로 통일합니다.
    - 공백 제거(strip)
    - 소수점 제거(예: `556.0` -> `556`)
    - 3자리 0 채우기(zfill, 예: `1` -> `001`)
*   **PRS ID 매핑**: `UserNo`가 없는 로그 데이터(예: 제안서 다운로드)는 `PRS ID`를 매개로 직원정보 마스터의 `UserNo`를 찾아 조인합니다.

### 2. 직원 소속 및 직급 매핑 규칙 (Employee Mapping)
*   **config.py 기반 동적 참조**: 모든 페이지(1~5)는 `config.py`에 정의된 연도별 컬럼 패턴 로직을 따릅니다.
    - 예: `{year}_부서명`, `{year}_통계 직급`
*   **부서 정보 Fallback (우선순위)**: 로그 발생 시점의 연도별 정보가 없거나 공란일 경우 다음 순서로 데이터를 찾습니다.
    1.  **부서명** (Dept)
    2.  **본부/실** (HQ)
    3.  **사업부** (Division)
*   **표준 컬럼 제공**: `data.py` 수준에서 위의 로직을 적용한 `부서`, `직급`, `직급그룹` 표준 컬럼을 생성하여 모든 대시보드가 동일한 정보를 보게 합니다.

### 3. 퇴사자 처리 방침 (Retirement Policy)
*   **활동 기록 보존**: 퇴사자라 하더라도 재직 당시의 활동 로그는 통계에 포함되어야 합니다.
    - **원칙**: 데이터 조인(Join) 시에는 `df_users` 마스터에서 어떠한 인원도 미리 필터링하지 않습니다.
    - **효과**: 과거 로그 로드 시 '정보미등록'이 발생하는 현상을 방지합니다.
*   **퇴사자 제외 시점**: 현재 인원수 집계나 '직원 명단' 조회 등 **'현재 시점'**의 정보가 필요한 경우에만 전용 시트(**`퇴사자_ignore`**)를 참조하여 필터링을 수행합니다.

### 4. 기타 예외 처리
*   **테스트 계정 제외**: `UserNo` 556 등 시스템 테스트용 계정은 모든 통계에서 명시적으로 제외합니다.
*   **데이터 정합성 필터**: 필터링 후에도 소속 정보가 없는 경우 최후의 수단으로만 '정보미등록'으로 표기합니다.
