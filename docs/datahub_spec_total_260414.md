# EZ데이터허브 대시보드 스펙 문서

> 버전: v5.0 | 최종 업데이트: 2026-04

---

## 1. 시스템 개요

### 1.1 목적

EZ데이터허브 임직원 사용 로그(로그인, 파일 다운로드, 제안서 다운로드)를 수집·분석하여 부서/직급별 사용 현황, KPI 달성 여부, 이상 사용 감지를 지원하는 내부 운영 대시보드.

### 1.2 기술 스택

| 구분 | 기술 |
|---|---|
| 프론트엔드/서버 | Streamlit (Python) |
| 데이터 저장소 | Google Sheets |
| 인증 | Google Service Account (gspread) |
| 배포 | Streamlit Community Cloud |
| 데이터 수집 | Python (requests, pandas, openpyxl) |
| 버전 관리 | GitHub (ezaxdx/datahub_log_dashboard) |

## 2. 데이터 파이프라인

### 2.1 수집

- 폐쇄망 인트라넷 API에서 PHPSESSID 세션 인증으로 로그 데이터 추출
- 수집 주기: 수동 실행 (GitHub Actions 자동화 불가 — 폐쇄망 특성)
- checkpoint 기반 신규 데이터만 필터링 (중복 적재 방지)
- 실행 시 로컬 Excel(.xlsx) 자동 백업

### 2.2 적재

- Google Sheets에 append_rows() 배치 처리로 누적 적재
- Sheets API 쿼터 초과 방지를 위해 배치 단위로 업로드

### 2.3 로드 및 전처리 (data.py)

`run_all()` 호출 시 아래 순서로 실행:

```
load_all()          Google Sheets에서 원본 데이터 로드 (캐시 ttl=600)
    ↓
exclude_users()     DEFAULT_EXCLUDE_USERNO 기준 제외 처리
    ↓
preprocess_all()    날짜 컬럼 통일, year 컬럼 추출
    ↓
map_all()           UserNo 정규화, PRS ID 역매핑, 연도 기반 마스터 조인
    ↓
add_rank_group()    직급 → 직급그룹 파생 컬럼 생성
```

### 2.4 UserNo 정규화 규칙

- 소수점 제거 (예: 123.0 → 123)
- 3자리 zero-padding (예: 5 → 005)
- 공백 제거

### 2.5 부서명 Fallback 우선순위

로그 데이터에서 부서명이 없는 경우 아래 순서로 대체:

```
1순위: {year}_부서명
2순위: {year}_본부/실
3순위: {year}_사업부
```

## 3. Google Sheets 시트 구성

### 3.1 직원정보 (SHEET_NAME_USERS)

2행 병합 헤더 구조 (1행: 연도, 2행: 컬럼명)

| 구분 | 컬럼 |
|---|---|
| 공통 | No, UserNo, 임직원명, PRS ID, 입사일자 |
| 2025년 | 소속, 부서명, 직위, 직급 |
| 2026년 | 사업부, 본부/실, 부서명, 직위, 직급, 통계 직급 |

> 현재 재직 임직원만 포함. 퇴사자는 퇴사자_ignore 시트로 분리 관리.

### 3.2 퇴사자_ignore (SHEET_NAME_IGNORE)

직원정보와 동일 구조 + 퇴사월 컬럼 추가. 데이터 로드 시 자동 제외 처리.

### 3.3 login (SHEET_NAME_LOGIN)

| 컬럼 | 설명 |
|---|---|
| NO, UserNo | 식별자 |
| 이름, 부서, 직급, 직책 | 사용자 정보 |
| IP, 브라우저, 디바이스, OS | 접속 환경 |
| 로그인 일자, 로그인 시간 | 접속 일시 |

### 3.4 download (SHEET_NAME_DOWNLOAD)

| 컬럼 | 설명 |
|---|---|
| NO, UserNo, PRS ID | 식별자 |
| 파일명, 크기, 경로, 경로 메뉴명 | 파일 정보 |
| 이름, 부서, 팀, 역할 | 사용자 정보 |
| 다운로드 일자, 다운로드 시간 | 다운로드 일시 |

> 경로 메뉴명 기준으로 프로젝트 찾기 / 운영자료 찾기 / 서포트 센터 구분.

### 3.5 제안서_ezPDF (SHEET_NAME_PROPOSAL)

| 컬럼 | 설명 |
|---|---|
| 번호, PRS ID | 식별자 |
| 문서경로 | 제안서 파일 경로 (프로젝트 코드 파싱 가능) |
| 등록일, 로그인 일자, 로그인 시간 | 열람 일시 |
| 이름, 팀, 역할 | 사용자 정보 |
| 성공 유/무, 유형, 유저ID, 비고 | 부가 정보 |

## 5. 페이지별 기능 스펙

### 5.1 app.py (메인)

- Streamlit 페이지 설정 및 전역 CSS 적용
- Google Sheets 데이터 최초 로드 및 세션 상태 저장
- 사이드바 구성: D+ 카운터, 메뉴 네비게이션, 상세 필터
- 사이드바 기본 필터값: DEFAULT_EXCLUDE_DEPTS + DEFAULT_EXCLUDE_USERNO 제외 상태로 초기화
- 선택된 페이지 exec() 방식으로 동적 실행

### 5.2 1_total.py (Total Dashboard)

**상단 KPI 카드 (5개)**
- 총 로그인, 제안서 DL, 프로젝트 찾기, 운영자료 찾기, 서포트 센터

**일자별 활동 현황**
- 로그인 수(좌축) + 다운로드 합계(우축) 이중 축 꺾은선 차트

**제안서 다운로드 경고 직원**
- warning_threshold 기준 초과 직원 테이블

**하단 TOP5 표 (1행 4열)**
- 부서별 로그인 TOP5
- 부서별 사용률 TOP5: 순사용자(f_proposal+f_download) / f_u 전체인원 * 100
- 직급별 로그인 TOP5
- 직급별 사용률 TOP5: 동일 계산식

### 5.3 2_File_Analysis.py (File Analysis)

**직원별 활동 상세내역**
- f_u 기준 전체 직원 × 로그인/제안서/프로젝트/운영자료/서포트 활동 건수 통합 테이블
- 제안서 다운로드 warning_threshold 초과 시 빨간색 강조

**제안서 경고 직원 리스트**
- warning_threshold 설정: 5/10/15/20/30/50/100건 선택
- 기준치 초과 직원 selectbox로 선택

**다운로드 타임라인 (슬라이딩 윈도우)**
- 전체 로그: 중복 문서 제외(UserNo+문서경로) 후 전체 표시
- N시간 선택: 첫 다운로드 시각 기준 슬라이딩 윈도우 적용
  - 윈도우 내 동일 문서 중복 제외 (문서경로 기준)
  - N시간내_순다운로드수 컬럼 추가
  - 다음 윈도우는 현재 윈도우 마지막 행 다음부터 시작
- 출력 컬럼: 이름, 부서, 직급, PRS ID, N시간내_순다운로드수, 윈도우시작, 문서이름, 열람시간

**다운로드 현황 TOP10 (1행 4열)**
- 제안서 TOP10: 문서경로에서 프로젝트 코드/명 파싱 (정규식: `/(\d{6})\[([^\]]+)\]`)
- 프로젝트 찾기 TOP10
- 운영자료 찾기 TOP10
- 서포트 센터 TOP10

### 5.4 3_department.py (Dept & Team)

**부서별 사용현황 막대차트**
- 로그 종류 선택: 로그인/제안서/다운로드 전체/프로젝트 찾기/운영자료 찾기/서포트센터
- 제외 그룹: M-Level(임원), 스마트관광 디지털융합혁신
- 막대 위 사용률 표시, 전사 평균 점선
- 사용률 = 순사용자 / f_u 부서_그룹 전체인원 * 100

**직급별 사용현황 막대차트 (임원 제외)**
- config.RANK_ORDER 기준 정렬
- 분모: f_u 기준 직급별 전체인원

**실무자/관리자 비중 도넛차트**

**부서/직급별 인원 현황 크로스탭**
- df_u 기준 부서_그룹 × 직급, 행 합계 포함

### 5.5 4_kpi.py (Check KPI)

- KPI: 데이터허브 사용률 30% 이상 증대 (자료 다운로드 1인당 평균 횟수 기준)

### 5.6 5_employee_list.py (Employee List)

- 사이드바 필터 미적용, 항상 df_u 전체 표시
- 표시 컬럼: UserNo, 이름, 부서, 직급, PRS ID, 입사일자
- CSV 다운로드 기능

## 6. 사용률 계산 기준

| 지표 | 분자 | 분모 | 비고 |
|---|---|---|---|
| 부서별 사용률 | f_proposal + f_download 순사용자 | f_u 부서별 전체인원 | 로그인만 한 사용자 제외 |
| 직급별 사용률 | f_proposal + f_download 순사용자 | f_u 직급별 전체인원 | 로그인만 한 사용자 제외 |
| 부서별 사용률 (Dept 페이지) | 선택 로그 기준 순사용자 | f_u 부서_그룹별 전체인원 | |

> 분모를 f_u 기준으로 통일한 이유: 사이드바 부서/직급 필터 적용 시 분모도 필터 범위 내 인원으로 정확하게 계산되어야 왜곡이 없음.

> 사용률 분자에서 로그인 제외 이유: KPI가 "자료 다운로드 1인당 평균 횟수" 기준이므로 다운로드 행위를 한 사용자만 분자에 포함.

## 8. 보안 및 인증

- Google Sheets 접근: 서비스 계정 JSON → `.streamlit/secrets.toml` 등록 (git 제외)
- `.gitignore`에 `secrets.toml`, `__pycache__/`, `.env` 포함
- DEFAULT_EXCLUDE_USERNO: 특정 사용자 데이터를 전체 파이프라인에서 제외 (data.py run_all 단계)
